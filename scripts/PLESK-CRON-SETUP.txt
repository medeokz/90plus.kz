WHY "File not found" on https://90plus.kz/plesk-deploy.php
=============================================================
1) Domain 90plus.kz not added in Plesk (only visitabay.kz works)
2) Code not uploaded / git pull not done
3) Wrong document root (httpdocs vs httpdocs/public)

STEP 1 — Find your real domain and path
---------------------------------------
Plesk → Files → find file "artisan"
Example paths (your server):
  /var/www/vhosts/018.kz/90plus.kz/artisan
  Document root is often the project root (OK-root on plesk-ping.php)

STEP 2 — Test document root in browser
---------------------------------------
Try BOTH (replace domain):

  https://90plus.kz/plesk-ping.php
  https://90plus.kz/public/plesk-ping.php

  https://visitabay.kz/plesk-ping.php
  https://visitabay.kz/public/plesk-ping.php

If you see "OK-public" → docroot = public/
If you see "OK-root" → docroot = httpdocs/
If "File not found" → upload files or fix domain DNS/hosting

STEP 3 — Deploy URL (after ping works)
---------------------------------------
If OK-public:
  https://YOUR-DOMAIN/plesk-deploy.php?key=YOUR_DEPLOY_KEY

If OK-root only:
  https://YOUR-DOMAIN/plesk-deploy.php?key=YOUR_DEPLOY_KEY

If only public/ works but URL without /public/ fails:
  Plesk → Hosting → Document root → set to httpdocs/public

STEP 4 — .env
---------------------------------------
DEPLOY_KEY=your-secret-same-as-in-url

STEP 5 — Plesk Scheduled Task (PHP script, NOT bash)
---------------------------------------
Type: Run a PHP script

If docroot = public:
  Script: /var/www/vhosts/018.kz/90plus.kz/public/plesk-deploy.php

If docroot = project root (OK-root on ping):
  Script: /var/www/vhosts/018.kz/90plus.kz/plesk-deploy.php
  Argument: your-secret

OR Fetch URL:
  https://YOUR-DOMAIN/plesk-deploy.php?key=your-secret

STEP 6 — Log file (created after first successful run)
---------------------------------------
httpdocs/storage/logs/cron-deploy.log
httpdocs/storage/logs/ping.log
